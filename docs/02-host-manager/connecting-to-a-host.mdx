# Connecting to a Host

Labonair handles the full connection lifecycle — from initiating the TCP connection to interactive authentication challenges — without leaving the current window.

## Starting a Connection

| Method | Result |
|---|---|
| Double-click a host card | Opens a new `ssh-terminal` tab |
| Right-click → **Connect SSH** | Opens a new `ssh-terminal` tab |
| Right-click → **Open SFTP** | Opens a new `sftp` tab |

Both SSH and SFTP connections go through the same Rust backend (`ssh2` crate), so the connection flow described below applies to both tab types.

## The Connection Loading Screen

When a connection is initiated, a **Loading Screen** replaces the terminal area in the new tab. It displays real-time status messages as the connection progresses:

1. Resolving hostname
2. Establishing TCP connection
3. SSH handshake
4. Authenticating
5. Opening session

## Authentication

If Labonair has a saved credential for this host (password or passphrase stored in Keychain), it attempts authentication automatically — no user interaction required.

If authentication fails, or if no credential is saved, the Loading Screen **transforms inline** into an authentication form:

- **Password prompt** — enter the password; an optional "Save password" checkbox stores it in Keychain for future connections.
- **Passphrase prompt** — shown when an SSH key is encrypted and no passphrase is saved.
- **2FA / Keyboard-Interactive prompt** — shown for TOTP codes or hardware key challenges. These are never saved.

Submitting the inline form retries the connection immediately — no need to restart the flow.

## Fingerprint Verification

On every connection, Labonair checks the server's SSH fingerprint against `~/.ssh/known_hosts`.

- **New host (no existing entry):** A "Trust this server?" dialog shows the server's fingerprint. Click **Trust & Connect** to add it to `known_hosts` and proceed.
- **Fingerprint mismatch:** The connection is blocked and a high-severity security warning is shown. See [known_hosts & Fingerprint Verification](../03-credential-management/known-hosts-fingerprint) for guidance.

## Successful Connection

When authentication succeeds, Labonair fires a `session_established` event internally, the Loading Screen disappears, and the terminal becomes fully interactive. For SFTP tabs, the remote file browser loads the home directory.

## Connection Errors

If the connection cannot be established (host unreachable, wrong port, auth rejected), the Loading Screen shows a descriptive error message and a **Retry** button. Common causes:

- Host firewall blocking port 22
- Incorrect username or credential
- SSH service not running on the server

:::tip
Use the terminal's built-in `ssh` command for one-off connections to hosts you don't want to save. The Host Manager is best suited for servers you connect to regularly.
:::
