# Security Model

Nexum is built around a strict separation between **host metadata** and **credentials**. The goal is simple: even if an attacker gains access to Nexum's database file, they cannot obtain any passwords, passphrases, or API keys.

## Two-Store Architecture

### SQLite — Host Metadata Only

Nexum stores the following in a local SQLite database at `~/.local/share/com.nexum.app/nexum.db`:

- Display name
- Hostname / IP address
- Port
- Username
- Authentication method (password, key, key+passphrase)
- Group assignment
- Host UUID
- Any notes or color labels

**No credentials of any kind are stored in SQLite.**

### macOS Keychain — Credentials Only

All secrets are stored in the macOS Keychain via Nexum's Rust `keyring` crate integration:

- SSH passwords
- SSH key passphrases
- AI provider API keys

Each credential is stored with:
- **Service**: `com.nexum.app`
- **Account**: the host's UUID (a random identifier, not the hostname)

The UUID-based account key means the Keychain entry reveals nothing about which server it belongs to, even if the Keychain is inspected directly.

## Why This Matters

If the SQLite database file is copied or leaked:

- The attacker sees hostnames, usernames, and connection metadata — essentially the same information stored in a plain-text SSH config.
- They cannot retrieve any password or passphrase, because those are locked inside the Keychain.

The Keychain itself is protected by macOS system security — unlocked only by the user's login password or Touch ID, and only accessible to processes that macOS has authorized (i.e., Nexum itself).

## Keychain Access Authorization

On the first credential save, macOS may display a prompt asking whether to allow Nexum to access the Keychain. Click **Always Allow** to prevent repeated prompts. If you accidentally click **Deny**, you can re-authorize by opening **Keychain Access.app**, finding the `com.nexum.app` entry, and updating its access control settings.

:::tip
To audit exactly what Nexum has stored, open **Keychain Access.app** and search for `com.nexum.app`. Each entry corresponds to one host's credential, identified by its UUID.
:::

:::warning
Nexum's Keychain entries are **not** synced to iCloud Keychain. They are local to the Mac where Nexum is installed. If you migrate to a new machine, you will need to re-enter credentials.
:::
