# Customizing Shortcuts

Labonair allows you to rebind any keyboard shortcut to a key combination of your choice. Custom bindings are stored persistently and survive app updates.

## Opening the Shortcuts Editor

There are two ways to open the Keyboard Shortcuts editor:

- Press `Cmd+?` from anywhere in the app
- Open Settings (`Cmd+,`) and navigate to **Keyboard Shortcuts**

The Shortcuts editor displays all available shortcuts organized by category (Global, Terminal, SFTP, Editor, Tab Management).

## Editing a Shortcut

1. Click any shortcut row to enter edit mode
2. Press your desired key combination — the new binding is shown in the input field
3. If the combination conflicts with an existing shortcut, the row is highlighted in red and a tooltip shows which action it conflicts with
4. Press `Enter` to save the new binding, or `Escape` to cancel without changes

You cannot save a binding that conflicts with another shortcut. Resolve the conflict first by rebinding the other shortcut to something else.

## Resetting Individual Shortcuts

Each shortcut row has a reset icon on the right side. Click it to restore that shortcut to its factory default. A brief confirmation appears before the reset is applied.

## Resetting All Shortcuts

A **Reset All** button at the top of the Shortcuts editor restores every shortcut to its default binding. A confirmation dialog appears before the reset is applied, as this action cannot be undone.

## Storage

Custom shortcuts are stored at:

```
~/.local/share/com.labonair.app/keybinds.json
```

This file is a plain JSON map of action ID to key combination. You can edit it directly in a text editor if you prefer, though the in-app editor is recommended to catch conflicts automatically.

:::tip
If you are migrating from another terminal emulator (iTerm2, Warp, Alacritty), start by checking which shortcuts conflict with your muscle memory and rebind them first before using Labonair for production work.
:::

:::warning
Deleting `keybinds.json` or the entire `~/.local/share/com.labonair.app/` directory resets all shortcuts to defaults along with all other app data. Back up the file separately if you want to preserve a custom shortcut layout.
:::
