# Code Editor Overview

Nexum ships a fully featured code editor built on **CodeMirror 6** (via `@uiw/react-codemirror`). It is not a stripped-down viewer — it is a proper editing surface designed to keep you in one window whether you are tweaking a local config file or adjusting a script on a remote server over SFTP.

## How it fits into the tab system

Every file you open creates a tab of type `editor`. Editor tabs live alongside terminal, SFTP, and SSH tabs in the same tab bar, so you can switch between a terminal session and the file you are editing with a single click. Multiple editor tabs can be open simultaneously — there is no artificial limit.

## Syntax highlighting for 50+ languages

Language detection is automatic. Nexum reads the file extension and maps it to the appropriate CodeMirror language pack via `languageResolver.ts`. Supported languages include TypeScript, JavaScript, Rust, Python, Go, Shell/Bash, JSON, YAML, TOML, HTML, CSS, Markdown, SQL, and many more. Language packs are loaded lazily, so only the grammar for the file you are actually viewing is fetched — startup time is not affected by the full language catalogue.

Syntax colors follow the active Nexum theme through injected CSS variables, so highlighting adapts automatically when you switch themes. No separate editor-theme setting is needed.

## Local and remote editing in one flow

The editor handles two distinct scenarios behind a consistent UI:

- **Local files** — opened directly from the filesystem; `Cmd+S` writes in place.
- **Remote files** — downloaded to a temp path, edited locally, then uploaded back via the active SFTP session on save. A cloud indicator in the tab signals the remote origin.

## AI integration

The AI assistant can read any open editor file as context, propose changes through the `write_file` tool, and open an **AI Diff** tab for your review before anything is written to disk. You accept, reject, or manually edit the proposal — the agent waits for your decision.

:::tip
You can have the AI summarize or refactor an open file by attaching it as a selection in the AI input bar. Highlight the relevant code in the editor, then use "Attach Selection" in the composer.
:::
