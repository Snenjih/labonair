# Live Context

The AI assistant is not operating in the dark. It has real-time access to two pieces of your workspace state: your **current working directory** and the **recent output of your active terminal**. This live context makes the assistant significantly more useful for infrastructure and DevOps work — it can see where you are and what you were just doing without you having to explain it.

## What Live Context Includes

### Current Working Directory (CWD)

The AI knows the directory your active terminal tab is currently in. This is tracked via **OSC sequences** — shell integration codes that Nexum's PTY layer injects into your shell's init script (`zshrc.zsh`, `bashrc.bash`, etc.). Every time you `cd` in the terminal, the CWD is updated in the frontend state automatically.

The AI uses CWD when resolving relative file paths in tool calls and when deciding where to create new files.

### Terminal Buffer (Last 300 Lines)

The AI can read the last 300 lines of scrollback from the currently active terminal tab. This is the same content you see on screen when you scroll up. The AI uses this to understand recent command output, error messages, build logs, and anything else that has scrolled past.

## Lazy Evaluation

Live context is read **lazily** — only at the moment a context-aware tool is actually called. Nexum does not continuously snapshot your terminal or pre-load context into every message. This means:

- No performance impact from having the AI panel open
- The context is always fresh — it reflects the state of your terminal at the moment the AI needs it, not when you started the conversation
- Switching terminal tabs before the AI processes your message gives the AI context from the tab you switched to

## Switching Tabs

Context is always derived from the **currently active terminal tab** in the Nexum workspace. If you have multiple SSH sessions open in different tabs, the AI sees the CWD and buffer of whichever tab is on screen when the tool runs.

If you want the AI to operate in the context of a specific session, make sure that tab is active before sending your message.

## Technical Setup

Live context is wired in `App.tsx` via:

```typescript
setLive({
  getCwd: () => activeTab?.cwd ?? null,
  getTerminalContext: () => activeTab?.terminalBuffer?.getLastLines(300) ?? [],
})
```

Tool implementations in `tools/tools.ts` call `getLive().getCwd()` and `getLive().getTerminalContext()` when they need workspace state. Both functions are synchronous and return the current snapshot at call time.

## Limitations

- Live context is only available from **local terminal tabs**. SSH terminal tabs (remote PTY sessions) do not currently feed their buffer into the live context system.
- The 300-line buffer limit means very verbose output (large log dumps, long `cat` output) may be truncated at the top. If you need the AI to analyze a long output, pipe it to a file and use `read_file` instead.
- Sub-agents do not receive live context — only the main agent loop has access.

:::tip
If the AI seems unaware of where you are or what you were just doing, check that you have an active local terminal tab — not an SFTP or editor tab — visible before sending your message. The live context bridge only activates when a terminal tab is the active workspace tab.
:::
