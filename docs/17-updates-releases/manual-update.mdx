# Manual Update

If the auto-updater doesn't trigger (for example, if automatic update checks are disabled, or if your network restricts access to GitHub), you can check for and apply updates manually at any time.

## Checking for Updates from Within the App

1. Press `Cmd+,` to open **Settings**.
2. In the Settings sidebar, click **About**.
3. Click the **"Check for Updates"** button.

Nexum will query the GitHub Releases endpoint immediately. If an update is found, the same "Update Available" dialog used by the auto-updater will appear, and you can proceed with the download and install from there.

## Downloading and Installing Manually

If you cannot use the in-app updater at all (managed device, air-gapped environment, or a known issue with the updater), you can update Nexum by reinstalling from a fresh download:

1. Go to the [Nexum GitHub Releases page](https://github.com/nexum-app/nexum/releases).
2. Find the latest release and download the appropriate `.dmg`:
   - **`Nexum_x.y.z_aarch64.dmg`** — Apple Silicon (M1/M2/M3/M4)
   - **`Nexum_x.y.z_x86_64.dmg`** — Intel Mac
3. Open the `.dmg` file.
4. Drag **Nexum.app** into your **Applications** folder.
5. When prompted by Finder, confirm **"Replace"** to overwrite the existing installation.

You do not need to uninstall the previous version first. The drag-and-replace operation is sufficient.

:::tip
Your hosts, settings, themes, API keys, and AI sessions are stored outside the app bundle — in `~/Library/Application Support/com.nexum.app/` and in the macOS Keychain. Replacing the `.app` bundle does not touch any of this data.
:::

## Verifying the Installed Version

After updating, confirm the version by opening **Settings** (`Cmd+,`) → **About**. The current version number is displayed at the top of that panel.

## Downgrading to an Older Version

Nexum does not enforce a minimum version restriction in settings, so downgrading is possible by installing an older `.dmg`:

1. Go to the [GitHub Releases page](https://github.com/nexum-app/nexum/releases).
2. Scroll to the desired older release or use the search filter.
3. Download the `.dmg` for that version.
4. Install it the same way as a manual update (drag and replace).

:::warning
Downgrading is generally safe for settings and host data, but use caution if the newer version migrated the database schema. A schema migration cannot always be reversed cleanly. If you downgraded and experience issues with hosts or settings loading, back up `~/Library/Application Support/com.nexum.app/` before proceeding and consult the release notes for migration notes.
:::
