# Creating Custom Themes

Building a custom Labonair theme requires nothing more than a text editor and a color picker. Themes are plain JSON files — no build step, no toolchain, no CSS knowledge required.

## Step-by-Step

### 1. Export the Default Theme

Open the Theme Picker (**Settings → Themes**), find **Default**, and click the Export icon. Save the file somewhere convenient — this gives you a complete, valid JSON starting point with both a `dark` and a `light` variant, every required key already present.

### 2. Open in a Text Editor

Open the exported `.json` file in any text editor. VS Code, Zed, Sublime Text, or even a plain editor works fine.

### 3. Set Name and Author

```json
{
  "name": "My Custom Theme",
  "author": "Your Name",
  "variants": {
    "dark": { "mode": "dark", "colors": { ... } },
    "light": { "mode": "light", "colors": { ... } }
  }
}
```

Change `name` to your theme's display name and add your name to `author`. Leave the `variants` keys (`dark`/`light`) as-is unless you're building a larger family (see the tip below) — each variant's `mode` field is what actually determines whether it's used in light or dark scheme, not the key name.

### 4. Edit the Colors

Work through **each variant's** `colors` map and replace HEX values with your chosen palette. A free online color picker like [coolors.co](https://coolors.co) or [oklch.com](https://oklch.com) makes it easy to explore palettes and copy HEX codes. A theme is only valid once both the `dark` and the `light` variant are filled in — a file with only one mode is rejected on import.

**Start with these high-impact keys, in both variants:**

- `background` — the main app background
- `foreground` — primary text color
- `primary` — your accent color (buttons, active tabs, selection highlights) — keeping this consistent across variants gives the theme a recognizable identity in both modes
- `border` — dividers and outlines
- `muted_foreground` — placeholder text and secondary labels

Then work through the `terminal.ansi.*` colors in each variant. These significantly affect day-to-day terminal use — choose colors with clear visual distinction between each slot.

### 5. Save as `.json`

Save the file with a `.json` extension. The filename can be anything — it does not need to match the `name` field.

### 6. Import into Labonair

In the Theme Picker, click **Import**, select your file, and confirm. If the file is valid, it appears in the list immediately and can be activated.

### 7. Iterate

Activate your theme, toggle the Color scheme setting to check both variants, look for contrast issues or jarring color combinations, tweak the JSON in your editor, and re-import. Because the import flow overwrites themes with the same filename, rapid iteration is straightforward.

### 8. Share

If you want to share your theme with the community, submit a pull request to the `labonair-themes` GitHub repository. Include your `.json` file and a short description.

:::tip
Keep sufficient contrast between `background` and `foreground` in **both** variants for comfortable reading. A contrast ratio of at least 4.5:1 is recommended (WCAG AA standard). Use a tool like [contrast ratio checker](https://webaim.org/resources/contrastchecker/) to verify your choices.
:::

:::tip
Building a larger family (like Catppuccin's Latte/Frappé/Macchiato/Mocha)? Add extra keys to `variants` beyond just `dark`/`light` — each still needs a `mode` of `"light"` or `"dark"`, and an optional `label` for the picker (e.g. `"label": "Macchiato"`). Labonair shows a small variant picker automatically whenever more than one variant shares the active mode.
:::
