# Exporting Themes

Every theme installed in Labonair — including the built-in "Labonair Dark" default — can be exported to a `.json` file. This is useful for sharing themes with colleagues, backing up a customized theme, or using an existing theme as a starting point for a new one.

## Export Steps

1. Open the Theme Picker via **Settings → Appearance → Theme** or `Cmd+K` → "Switch Theme".
2. Hover over the theme you want to export. An **Export** icon (download arrow) appears on the right side of the theme tile.
3. Click the Export icon.
4. An OS-native save dialog opens. Choose a destination folder and filename, then click **Save**.
5. The theme JSON is written to the selected location.

## What Gets Exported

The exported file is the complete theme JSON as stored internally by Labonair — it includes the `name`, `author` (if set), `type`, and the full `colors` map with all UI and terminal ANSI color keys. The exported file is immediately valid for re-import into Labonair or sharing with others.

## Exporting the Default Theme

The built-in "Labonair Dark" theme can be exported exactly like any other theme. This is the recommended way to start building a custom theme — export the default, open it in a text editor, and modify the colors. See [Creating Custom Themes](./creating-custom-themes) for a full walkthrough.

## Technical Details

The export flow is handled by the `theme_export` Tauri IPC command on the Rust backend. When invoked, it opens a native save dialog via the Tauri `dialog` plugin and writes the JSON file using Rust's `std::fs`. The webview has no direct filesystem access.

:::tip
Use a descriptive filename when exporting — the filename does not need to match the theme's `name` field, but matching them keeps things organized. Example: `catppuccin-mocha.json`.
:::
