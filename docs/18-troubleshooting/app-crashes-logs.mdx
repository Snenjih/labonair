# App Crashes & Logs

When Labonair crashes or behaves unexpectedly, the following resources help diagnose the cause.

## Crash Logs

macOS stores crash reports automatically for all applications. Look for Labonair crash reports at:

```
~/Library/Logs/DiagnosticReports/
```

Filter for files matching `Labonair-*.ips`. These `.ips` files contain the full crash stack trace, OS version, and memory state at the time of the crash. Attach the relevant `.ips` file when filing a bug report.

You can also view crash reports in **Console.app**:
1. Open Console.app (Applications → Utilities → Console)
2. Select "Crash Reports" in the left sidebar
3. Filter for "Labonair"

## Tauri / Rust Backend Logs

To see Rust backend log output (Tauri events, errors from the SSH/SFTP subsystem), launch Labonair from Terminal:

```bash
open -a Labonair
```

Stderr and log output from the Rust backend will print to the Terminal session. Alternatively, filter by "Labonair" in Console.app's live log stream.

## WKWebView Crash ("WebContent Process Terminated")

Labonair renders its UI in a WKWebView (Apple's WebKit renderer). Occasionally the WebContent subprocess — which is a separate OS process from Labonair itself — can crash. This is a WebKit renderer crash, not a Labonair crash.

**Symptoms:** The UI goes blank; the window may disappear briefly.

**Labonair's behavior:** Labonair detects the WebContent process termination event and automatically attempts to show the window again. Running terminal and SSH sessions in the Rust backend are not affected.

**If it happens repeatedly:** Reduce the number of open tabs. Large numbers of xterm.js instances with WebGL rendering are the most common cause of WebContent memory pressure.

## Reporting Bugs

File bugs at the Labonair GitHub repository. Include:
- A description of what you were doing when the crash occurred
- The `Labonair-*.ips` crash report file
- macOS version (`sw_vers`)
- Labonair version (Help menu → About Labonair)

## Resetting App Data

If Labonair is in a broken state that persists across restarts, resetting all app data is the nuclear option:

```bash
rm -rf ~/.local/share/com.labonair.app/
```

:::warning
This permanently deletes all saved hosts, custom themes, settings, and keyboard shortcut customizations. There is no recovery path. Export your hosts and themes before doing this.
:::
